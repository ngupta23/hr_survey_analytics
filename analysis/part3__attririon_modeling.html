Introduction¶

This notebook loads the data from the R script (which contains the information from the surveys and demographics combined into a single table). Using this data, we try to build several models to predict the likelihood of an employee leaving voluntarily in the coming months.

The primary goal is to use the output of the model to identify employees that are at risk of leaving and build targetted retention plan or packages for them. This plan could include things like long term stock incentives, possibility of career growth throgh promotion or more responsibilities. It could further be fine tuned based on whether the empployee is high performance/high potential or not. Foe example, if a low perfomance/low potetial employee is likely to leave voluntarily, a retention program may not be needed.

By proactively identifying employees likely to leave voluntarily and creating an action plan to retain them, the compnay could save a substantial amount of money in the long run.

Model Development¶

This section includes some technical details about how the model(s) were developed. Skip to the Conclusion section for summary.

The Dataset is highly imbalanced. Initially, the model was built with the original data "as-is" with only a balanced weight used in the model development. But with this approach, the models were overpredicting the likelihood of non attrition so it did not meet the original objective.

After various combinations of techniques, the one that gave the best results was the one that oversampled the "attrition" class level (using SMOTE) just for the training. This gave a better result that approaches without oversampling.

Two (2) models were tried for this initial analysis - Support Vector Machines and Random Forest Classifiers. This included a baseline model with default parameters and a tuned model with hyperparameter optimization. The best results were obtained using the Baseline Support Vector Machine model.

Conclusion¶

While the best model developed performed better than random guessing, it is not good enough yet to put into production. The model still only correctly predicts 1 in 3 people that will leave and overestimates 5 employees that will leave (but dont) for every 1 employee that is correctly identified as leaving. Nonetheless this is an important step in the roadmap to reduce attrition. Some ways to improve the model performance are suggested below.

  • The dataset did not contain some important features that can impact attrition such as pending incentives that the employee has. Getting access to this data would likely improve the model's performance.
  • The output of the model should not be the sole point of reference that is used to decide whether an employee is likely to leave. This should serve as one reference point, and other reference points must be gathered to triangualte on the employees that are actually likely to leave. For example, the model incorrectly predicts that 5 employees will leave for every 1 employee that actually leaves. However, some of these 5 employees could be eliminated from the list of high attrition employees (for example, by looking at employees that have high pending stock incentives), then we could narrow down the list to more likelt candidates.

# set up hyperparameter space
space = [
    Real(1e-5, 1e-0, name='alpha', prior='log-uniform'),
    Real(10e2, 10e6, name='max_iter',prior='log-uniform'), # needs an integer but that does not have log-normal sampling. Hence using Real (algorith works with real as well)
    Integer(3,7,name='n_iter_no_change')
]

# Object to scale the inputs before training
std_scl = StandardScaler()

@use_named_args(space)
def objective(**params):
    svm_sgd = SGDClassifier(learning_rate = 'optimal'
                            #,class_weight = 'balanced'
                            ,class_weight=class_weight
                            ,loss = 'hinge'
                            ,n_jobs = -1
                            ,random_state = 101
                            ,**params)
    
    piped_object = Pipeline([('scale', std_scl),  # do this
                             ('svm_model', svm_sgd)]) # and then do this

    # Get all metrics and define custom loss function using this data
    cv_results = cross_validate(piped_object, X_train, y_train, cv=cv_object, scoring = scoring)
    # loss = -np.mean(cv_results['test_recall_promoted_only'] + cv_results['test_f1_weighted'])
    # loss = -np.mean(cv_results['test_recall_attrited_only'] + cv_results['test_f1_attrited_only'])
    loss = -np.mean(
        cv_results['test_recall_attrited_only'] + \
        cv_results['test_precision_attrited_only'] 
    )
   
    return (loss)

# set up hyperparameter space
space = [Integer(100, 1000, name='n_estimators')#, prior='log-uniform')
         ,Integer(10, 1000, name='max_depth')#, prior='log-uniform')
         ,Real(0.000001, 0.1, name='min_samples_split', prior='log-uniform')
         ,Real(0.000001, 0.1, name='min_samples_leaf', prior='log-uniform')
        ]

@use_named_args(space)
def objective(**params):
    rf_std  =RandomForestClassifier(n_jobs=-1
                                   , random_state = 101 
                                   # , class_weight='balanced'
                                   , class_weight=class_weight
                                   ,**params)
    
    piped_object = Pipeline([('rf_model', rf_std)])  

    # Get all metrics and define custom loss function using this data
    cv_results = cross_validate(piped_object, X_train, y_train, cv=cv_object, scoring = scoring)
    # loss = -np.mean(cv_results['test_recall_promoted_only'] + cv_results['test_f1_weighted'])
    # loss = -np.mean(cv_results['test_recall_attrited_only'] + cv_results['test_f1_attrited_only'])
    loss = -np.mean(
        cv_results['test_recall_attrited_only'] + \
        cv_results['test_precision_attrited_only'] 
    )
   
    return (loss)
